# 포트폴리오 사이트 PRD

## 1. 문서 정보

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 박다효 포트폴리오 사이트 |
| 버전 | v0.1 |
| 작성일 | 2026-06-19 |

## 2. 목적

박다효의 업무 경험과 프로젝트 수행 내용을 웹 포트폴리오 형태로 정리한다.

사이트는 단순한 작품 전시가 아니라, **"무엇을 했는가(What)"**와 **"어떻게 했는가(How)"**를 분리해 보여주는 구조를 가진다.

### 콘텐츠 원본

- **What**: [기획_포트폴리오_박다효.pdf](https://drive.google.com/file/d/1xRLjLQ0LX5LlSQL9Cg8YCZDN1MgZnRFL/view?usp=sharing) 및 [Figma 파일](https://www.figma.com/design/UB4yqHy2WuvoVduPbunGw5/%ED%8F%AC%ED%8F%B4%EC%9A%A9---%EA%B3%84%EC%86%8D-%EC%82%AC%EC%9A%A9?node-id=18-1074&t=OmK9y0t17ZlE5QQw-1)의 프로젝트 내용
- **How**: 추후 노션으로 정리한 프로젝트별 상세 수행 과정

## 3. 타겟 사용자

- 채용 담당자 / 면접관
- 협업 파트너
- 포트폴리오를 빠르게 훑어보고 싶은 방문자

## 4. 페이지 구성

### 4.1 Home (`/`)

**역할**
- 간단한 자기소개
- 이력 요약
- 주요 역량 소개
- 포트폴리오(What) 페이지로 진입하는 링크 제공

**포함 섹션**
- Hero: 이름, 한 줄 소개, 연락처/링크
- About: 2~3문단 자기소개
- Experience: 회사/기간/역할 bullet
- Skills: 기술·도구 태그

### 4.2 What (`/portfolio/what`)

**역할**
- PDF/Figma 3번 페이지 목차 내용을 리스트 게시판 형태로 재구성
- 프로젝트별 아코디언 UI 제공
- 각 프로젝트에서 "무슨 업무를 했는지"를 빠르게 확인

**UI 형태**
- 리스트 게시판형 목차 (카드형이 아닌 목록형)
- 프로젝트 1개 = 아코디언 1개
- 접힌 상태: 목록에서 훑어보는 최소 정보
- 펼친 상태: 프로젝트 요약 상세 + How 상세 링크

### 4.3 How (`/portfolio/how/[slug]`)

**역할**
- What의 각 아코디언과 연결되는 상세 설명 페이지
- 프로젝트를 어떤 방식으로 수행했는지 상세 설명
- 초기에는 페이지 틀만 제공, 상세 내용은 추후 노션 정리본 기반으로 작성

**초기 상태**
- 프로젝트명, 기간, 역할 표시
- "상세 내용 준비 중" 안내
- What 페이지로 돌아가기 링크

**추후 확장 섹션**
- 문제 정의
- 맡은 역할 (상세)
- 진행 과정
- 사용한 도구/방법
- 협업 방식
- 결과 및 회고

## 5. What 아코디언 상세 스펙

### 5.1 접힌 상태

| 필드 | 설명 | 예시 |
|------|------|------|
| 번호 | 목차 순서 | `01` |
| 프로젝트명 | 가장 크게 보이는 제목 | `결제 프로세스 개선` |
| 기간 | 프로젝트 진행 기간 | `2024.01 - 2024.03` |
| 역할 | 담당 역할 | `UX/UI Designer` |
| 한 줄 요약 | 수행 업무 1문장 요약 | `결제 이탈을 줄이기 위해 주문부터 완료까지의 흐름을 개선한 프로젝트` |
| 태그 | 키워드 | `#UX` `#Figma` `#기획` |

### 5.2 펼친 상태

| 섹션 | 설명 | 권장 분량 |
|------|------|-----------|
| 프로젝트 개요 | 서비스/업무 맥락 설명 | 2~3문장 |
| 담당 업무 | 맡은 업무 bullet 목록 | 3~5개 |
| 주요 산출물 | 결과물 목록 | 3~5개 |
| 핵심 포인트 | 대표 키워드/성과 | 2~4개 |
| How 링크 | 상세 페이지 이동 | 버튼 1개 |

### 5.3 길이 가이드

- 한 줄 요약: 1문장
- 프로젝트 개요: 2~3문장
- 담당 업무: 3~5개 bullet
- 주요 산출물: 3~5개 bullet
- 핵심 포인트: 2~4개
- 펼친 상태 전체: 스크롤 1화면 안팎

## 6. 사용자 흐름

```
Home (/) 진입
  → 자기소개 / 이력 확인
  → What (/portfolio/what) 이동
  → 프로젝트 목록 훑기
  → 관심 프로젝트 아코디언 펼치기
  → 프로젝트 개요 / 담당 업무 확인
  → "How 자세히 보기" 클릭
  → How (/portfolio/how/[slug]) 상세 페이지
```

## 7. 핵심 요구사항

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| REQ-01 | What과 How는 동일한 `slug`로 연결 | 필수 |
| REQ-02 | What은 목록 탐색에 최적화 (게시판형 리스트) | 필수 |
| REQ-03 | What 아코디언 접힘/펼침 동작 | 필수 |
| REQ-04 | How는 상세 설명 페이지로 분리 | 필수 |
| REQ-05 | How 초기 상태에서 "준비 중" 표시 가능 | 필수 |
| REQ-06 | 모바일에서 아코디언 자연스럽게 동작 | 필수 |
| REQ-07 | PDF/Figma 원본 프로젝트 순서 유지 | 필수 |
| REQ-08 | How 콘텐츠는 노션 정리본 기반으로 추후 연동 | 추후 |

## 8. 비기능 요구사항

- 반응형 디자인 (모바일 우선 고려)
- 정적 사이트 배포 가능 (Vercel / Netlify 등)
- SEO 기본 설정 (title, description, OG)
- 콘텐츠는 코드 외부 파일(YAML/MD)로 관리
