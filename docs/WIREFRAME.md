# 포트폴리오 사이트 와이어프레임

## 1. 전체 사이트맵

```
/                          Home (자기소개 + 이력)
/portfolio/what            What (프로젝트 목차 아코디언)
/portfolio/how/[slug]      How (프로젝트별 상세)
```

## 2. 공통 레이아웃

### Header

```
┌──────────────────────────────────────────────────────────┐
│  박다효                              About   What   How  │
└──────────────────────────────────────────────────────────┘
```

- 좌측: 이름 (Home 링크)
- 우측: 네비게이션 (About → `/`, What → `/portfolio/what`)
- How는 개별 slug 페이지이므로 네비에 직접 링크하지 않음

### Footer

```
┌──────────────────────────────────────────────────────────┐
│  © 2026 박다효  ·  이메일 / 노션 / 기타 링크             │
└──────────────────────────────────────────────────────────┘
```

### 4.2 아코디언 — 접힌 상태

```
┌──────────────────────────────────────────────────────────┐
│ 01  결제 프로세스 개선                              [+]  │
│     2024.01 - 2024.03  ·  UX/UI Designer                 │
│     결제 이탈을 줄이기 위해 주문부터 완료까지의          │
│     흐름을 개선한 프로젝트                               │
│     #UX  #Figma  #기획                                   │
├──────────────────────────────────────────────────────────┤
│ 02  어드민 대시보드 리뉴얼                          [+]  │
│     2023.06 - 2023.12  ·  Product Designer               │
│     운영 효율을 높이기 위한 관리자 화면 재설계           │

├──────────────────────────────────────────────────────────┤
│ 03  브랜드 사이트 구축                              [+]  │
│     ...                                                  │
└──────────────────────────────────────────────────────────┘
```

**접힌 상태 표시 정보**
- 번호 (order)
- 프로젝트명 (title)
- 기간 (period) · 역할 (role)
- 한 줄 요약 (summary)
- 태그 (tags)
- 펼치기 아이콘 `[+]`

**펼친 상태 표시 정보**
- 접힌 상태 정보 (동일)
- 프로젝트 개요 (overview)
- 담당 업무 (responsibilities) — bullet list
- 주요 산출물 (deliverables) — bullet list
- 핵심 포인트 (highlights) — 태그/칩 형태
- `How 자세히 보기` 버튼 → `/portfolio/how/[slug]`

### 4.4 아코디언 동작 규칙

| 동작 | 설명 |
|------|------|
| 클릭 시 토글 | 접힘 ↔ 펼침 전환 |
| 다중 열기 | 허용 (여러 프로젝트 동시 펼침 가능) |
| How 링크 | howStatus가 `draft`여도 링크는 활성, 상세 페이지에서 "준비 중" 표시 |
| howStatus 배지 | `draft`일 때 "How 준비 중" 작은 배지 표시 (선택) |

## 6. 모바일 레이아웃

### What 아코디언 (모바일)

```
┌────────────────────────┐
│ Header (햄버거 메뉴)   │
├────────────────────────┤
│ Portfolio / What       │
│ 설명 텍스트            │
├────────────────────────┤
│ 01                     │
│ 결제 프로세스 개선 [+] │
│ 2024.01 - 2024.03      │
│ UX/UI Designer         │
│ 한 줄 요약...          │
│ #UX #Figma             │
├────────────────────────┤
│ 02                     │
│ 어드민 대시보드... [+] │
│ ...                    │
└────────────────────────┘
```

- 모든 정보가 세로 1컬럼
- 펼친 내용도 1컬럼
- 핵심 포인트 칩은 줄바꿈 허용
- `How 자세히 보기` 버튼은 full-width

### How 상세 (모바일)

- 상단 "What으로 돌아가기" 링크
- 제목, 기간, 역할
- 섹션별 세로 스택
- 좌우 패딩 확보

## 7. 컴포넌트 목록

| 컴포넌트 | 사용 위치 | 설명 |
|----------|-----------|------|
| `Header` | 전체 | 로고 + 네비게이션 |
| `Footer` | 전체 | 저작권 + 링크 |
| `Section` | Home, How | 섹션 래퍼 (제목 + 내용) |
| `Tag` | What, Home | 태그/칩 UI |
| `Accordion` | What | 아코디언 컨테이너 |
| `AccordionItem` | What | 개별 프로젝트 아코디언 |
| `AccordionTrigger` | What | 접힌 상태 (클릭 영역) |
| `AccordionContent` | What | 펼친 상태 내용 |
| `HighlightChips` | What | 핵심 포인트 칩 목록 |
| `HowLink` | What | How 상세 이동 버튼 |
| `BackLink` | How | What으로 돌아가기 |
| `ComingSoon` | How | 준비 중 안내 |
| `ExperienceCard` | Home | 이력 항목 |
