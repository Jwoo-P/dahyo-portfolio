## 프로젝트 개요

한국문화정보원의 실감데이터 공동활용 플랫폼에서 공공기관·체험시설이 제공하는 문화 데이터를 누구나 쉽게 활용할 수 있도록, 접근성과 사용성을 개선하는 고도화 프로젝트입니다.

![실감데이터 공동활용플랫폼 — 플랫폼 목적과 프로젝트 목표](/images/how/immersive-data-platform/01-purpose.jpg)

## 문제 정의

기존 플랫폼은 실감콘텐츠·실감데이터 용어가 낯설고, 메인 페이지 통합 검색 부재, 다운로드·회원가입 과정의 불편함, 관리자 페이지 접근 경로 혼란 등 사용성 문제가 있었습니다. 퀄리티 있는 문화 데이터를 쉽게 활용·관리할 수 있도록 개선이 필요했습니다.

## 맡은 역할

- 화면 기획
- 테스트 시나리오 작성
- Focus Group Interview 진행

## 진행 과정

### FGI 및 요구사항 정리

공공기관·체험시설 이용자와 일반 이용자 두 그룹으로 나누어 ZOOM 화상 채팅 방식의 FGI를 진행했습니다. 플랫폼 주요 기능 소개, 그룹별 테스트 시나리오 제공, 설문, Q&A 순으로 구성했습니다.

![FGI 진행 및 요구사항·개선사항 정리](/images/how/immersive-data-platform/02-fgi.jpg)

수렴된 의견을 바탕으로 메인 페이지 구성 재정비, 기술 용어 설명 추가, 다운로드 프로세스 개선, 회원가입·로그인 프로세스 변경, 메뉴 구조 재정비, 관리자 페이지 접근 경로·기능 개선, 사전 협의 데이터 관리 기능 도입 등을 정리했습니다.

### 메뉴 구조 재설계

사용자·기관 사용자·관리자 메뉴 구조를 재정비했습니다. 실감데이터 자원의 분류 기준을 변경하고, 콘텐츠·데이터 등록·관리 메뉴를 분리하여 혼란을 줄였습니다.

![사용자·기관·관리자 메뉴 구조 재설계](/images/how/immersive-data-platform/03-menu-structure.jpg)

### 사용자 화면 TO-BE

메인 페이지 통합 검색, 용어 설명 추가, 테마별 직관적 분류를 반영했습니다. 실감데이터 목록에서는 1,223건을 조사하여 10가지 유형의 상세 정보를 제공하고, 다운로드 시 여러 파일 형식을 한 번에 선택할 수 있도록 체크박스 방식으로 개선했습니다.

![사용자 TO-BE — 메인·목록·상세·다운로드](/images/how/immersive-data-platform/04-user-tobe.jpg)

### 관리자 화면 TO-BE

데이터별 사업명 구분, 엑셀 양식 다운로드·업로드를 통한 일괄 등록, 사전 협의 통계에서 진행률 확인 기능을 설계했습니다.

![관리자 TO-BE — 데이터 등록 및 진행률 모니터링](/images/how/immersive-data-platform/05-admin-tobe.jpg)

## 사용한 도구/방법

- Focus Group Interview (ZOOM 화상)
- 테스트 시나리오 작성
- 요구사항·개선사항 정리 및 화면 설계

## 협업 방식

피씨엔 프로젝트로 기획·디자인·개발 팀과 협업했습니다. FGI 진행과 테스트 시나리오 작성, 사용자·관리자 화면 기획을 담당했으며, 2회 감리를 통과할 수 있도록 요구사항 기반으로 화면을 설계했습니다.

## 결과 및 회고

홈페이지 개선 후 1,359명 대상 만족도 설문에서 UI/UX 만족도, 검색 용이성, 다운로드 편의성 항목 모두 평균 88%의 긍정 평가를 받으며 프로젝트를 마무리했습니다.

![사용자 만족도 평가 결과](/images/how/immersive-data-platform/06-satisfaction.jpg)
