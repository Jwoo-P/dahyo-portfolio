## 프로젝트 개요

원활한 자전거 공급을 위한 따릉이 대여 과정 개선 프로젝트입니다. Play Store 리뷰 등 데스크 리서치로 이용자 불편을 정의하고, 예약·목적지 설정 기능을 중심으로 User flow와 UI를 개선했습니다.

![프로젝트 메인 시안](/images/how/ddareungi-redesign/01-main.jpg)

## 문제 정의

거치대에 자전거가 없어 빌리지 못한 경험이 반복적으로 제기되었습니다. 이용권이 있어도 원하는 시간·장소에서 대여하지 못하거나, 앱에 표시된 재고와 실제 수량이 맞지 않는 경우가 많았습니다.

![거치대 재고 부족 관련 이용자 리뷰](/images/how/ddareungi-redesign/02-overview.jpg)

## 맡은 역할

- UI/UX 디자인 100%
- 데스크 리서치 기반 문제 정의 및 개선 방향 도출
- AS-IS / TO-BE User flow 설계
- 주요 화면 UI 시안 제작

## 진행 과정

적재적소에 공급할 수 있도록, 사용자가 원하는 시간대·거치대의 수요를 파악하는 것을 목표로 잡았습니다. 예약 기능과 목적지 거치대 설정을 추가해 거치대별·시간별 수요와 반납 예상 재고를 수집할 수 있도록 User flow를 확장했습니다.

![프로젝트 목표 및 AS-IS / TO-BE flow](/images/how/ddareungi-redesign/03-goal.png)

로그인 여부, 이용권 보유 여부, 거치대 선택 상태에 따라 노출 버튼과 이동 경로가 달라지도록 전체 흐름을 정리했습니다.

![리디자인 대상 User flow](/images/how/ddareungi-redesign/04-flowchart.png)

메인 화면에서는 자전거가 있는 거치대를 빠르게 파악하고, 선택한 거치대 현황을 명확히 보이도록 필터·마커·하단 정보 카드를 개선했습니다.

![메인 화면 AS-IS / TO-BE 비교](/images/how/ddareungi-redesign/06-main-as-is-to-be.png)

예약 화면에서는 시간대별 예약 현황을 보여주고, 예약 변경·취소까지 한 흐름으로 설계했습니다.

![예약 기능 UI](/images/how/ddareungi-redesign/07-reservation.png)

대여 후 블루투스 연결 대기 시간에 목적지 거치대를 입력하도록 유도했습니다. 검색·지도 선택 모두 가능하며, 건너뛰기도 지원해 사용성 부담을 줄였습니다.

![목적지 거치대 설정 — 대여 중 입력 유도](/images/how/ddareungi-redesign/08-destination-setup.png)

![목적지 거치대 설정 — 검색 및 선택](/images/how/ddareungi-redesign/09-destination-search.png)

## 사용한 도구/방법

- 데스크 리서치: Play Store '서울자전거 따릉이' 리뷰 분석
- UX 리서치 이론 기반 문제 정의
- User flow / flowchart 설계
- Figma UI 시안 제작

![Style guide](/images/how/ddareungi-redesign/09-style-guide.png)

## 협업 방식

개인 프로젝트로 진행했습니다. 리서치부터 flow 설계, UI 시안까지 단독으로 수행했습니다.

## 결과 및 회고

이용권 구매·대여 과정의 TO-BE UI 시안과 함께, 수요 파악을 위한 예약·목적지 설정 기능을 제안했습니다. 실제 서비스에 적용되기 전 단계의 개선안이지만, 이용자 리뷰에서 반복된 불편을 구조적으로 풀어보는 연습이 되었습니다.
