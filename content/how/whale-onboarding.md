## 프로젝트 개요

유저 리서치와 디자인 시스템을 기반으로 네이버 웨일 웹 온보딩 UX를 개선한 팀 프로젝트입니다. 인터랙션과 시각적 요소를 활용해 웨일만의 차별화된 브라우저 기능을 전달하는 온보딩 경험을 설계했습니다.

![네이버 웨일 웹 온보딩 UX 개선 프로젝트](/images/how/whale-onboarding/01.png)

웨일은 캡처, 듀얼 탭, 사이드바, 퀵서치 등 다양한 차별화 기능을 제공하지만, 기존 온보딩에서는 이러한 기능이 충분히 소개되지 않는다는 문제의식에서 출발했습니다.

![프로젝트 Overview — 웨일의 차별화 기능과 온보딩의 한계](/images/how/whale-onboarding/02.png)

## 문제 정의

AS-IS 온보딩을 분석한 결과, 친근한 톤은 유지되지만 배경 그래픽이 콘텐츠 집중을 방해하고, 웨일만의 핵심 기능 안내가 부족하다는 점이 확인되었습니다.

![AS-IS 온보딩 분석](/images/how/whale-onboarding/03.png)

가설 검증과 유저 인터뷰를 통해 문제를 4가지로 정의했습니다. 차별화 기능 소개 부족, 흥미·집중 요소 부족, 시선이 분산되는 레이아웃, 이해를 돕지 않는 시각적 설명 부족이 핵심 과제였습니다.

![문제 정의 — 4가지 핵심 과제와 해결 방향](/images/how/whale-onboarding/09.png)

## 맡은 역할

- UI/UX 디자인
- 유저 리서치 기반 문제 정의 및 UX 컨셉 도출
- 페르소나·UX 시나리오 설계
- 디자인 시스템 및 TO-BE 온보딩 UI 시안 제작

## 진행 과정

### 리서치 및 가설 검증

초기 가설 4가지를 설정하고, Opera·Vivaldi·Arc 등 타 브라우저 온보딩을 벤치마킹하며 사용자 검증이 필요한 요소를 분류했습니다.

![가설 검증 — 초기 가설과 벤치마킹 요소 발견](/images/how/whale-onboarding/04.png)

![타 브라우저 온보딩 벤치마킹](/images/how/whale-onboarding/05.png)

설문조사와 유저 인터뷰를 병행해 정성·정량 데이터를 수집했고, 인터뷰 인사이트를 바탕으로 개선 방향을 구체화했습니다.

![유저 리서치 — 설문조사](/images/how/whale-onboarding/06.png)

![유저 인터뷰 — 정성 리서치](/images/how/whale-onboarding/07.png)

![유저 인터뷰 — 정량 리서치](/images/how/whale-onboarding/08.png)

### UX 컨셉 및 목표 설정

리서치 결과를 바탕으로 차별화된 기능 안내, 역동적인 플로우, 집중되는 레이아웃, 직관적인 시각적 요소 4가지 UX 컨셉을 도출했습니다.

![UX 컨셉 도출](/images/how/whale-onboarding/10.png)

![프로젝트 목표](/images/how/whale-onboarding/11.png)

### 페르소나 및 UX 시나리오

두 가지 페르소나를 설정하고, 다운로드부터 메인 화면 진입까지의 온보딩 여정에서 각 단계별 사용자 가치를 정의했습니다.

![페르소나 1](/images/how/whale-onboarding/12.png)

![페르소나 2](/images/how/whale-onboarding/13.png)

![UX 시나리오](/images/how/whale-onboarding/14.png)

### 디자인 시스템

온보딩 전반에 일관된 톤앤매너를 적용하기 위해 컬러, 타이포그래피, 컴포넌트 가이드를 정리했습니다.

![UI 디자인 가이드](/images/how/whale-onboarding/15.png)

![UI 디자인 가이드 — 컴포넌트](/images/how/whale-onboarding/16.png)

### TO-BE 온보딩 UI

다운로드 팝업과 일관된 우주 테마 배경으로 몰입감을 높이고, 콘텐츠 대비를 강화해 시선이 분산되지 않도록 인트로 화면을 개선했습니다.

![인트로 페이지 — AS-IS / TO-BE 비교](/images/how/whale-onboarding/17.png)

타 브라우저 정보 가져오기 화면은 드롭다운 대신 세로형 리스트로 변경해 한눈에 옵션을 확인할 수 있게 했고, '건너뛰기'를 '나중에 하기'로 수정해 의도를 명확히 했습니다.

![브라우저 정보 가져오기](/images/how/whale-onboarding/18.png)

스킨 컬러 선택 단계에서는 실제 브라우저 프리뷰와 함께 컬러 팔레트를 보여주어 개인화 설정을 직관적으로 이해할 수 있도록 했습니다.

![웨일 스킨 컬러 선택](/images/how/whale-onboarding/19.png)

검색 사이트 설정과 주소창 검색 기능을 한 화면에 하나씩 소개하는 방식으로 정보 전달량을 조절했습니다.

![주소창 검색 설정](/images/how/whale-onboarding/20.png)

사이드바 탐색은 위젯 아이콘에 마우스를 호버하면 실제 화면 GIF가 재생되어, 기능을 빠르고 쉽게 이해할 수 있도록 설계했습니다.

![사이드바 위젯 탐색](/images/how/whale-onboarding/21.png)

검색 엔진 설정 등 나머지 온보딩 단계도 동일한 디자인 시스템과 인터랙션 원칙을 적용했습니다.

![검색 설정](/images/how/whale-onboarding/22.png)

## 사용한 도구/방법

- UX 리서치: 설문조사, 유저 인터뷰(정성·정량)
- 가설 검증 및 타 브라우저 벤치마킹 (Opera, Vivaldi, Arc 등)
- 페르소나·UX 시나리오 설계
- Figma UI 시안 및 디자인 시스템 제작

## 협업 방식

제로베이스 1기 팀 프로젝트로 진행했습니다. 네이버 웨일 기획팀과 협업하며 포트폴리오 리뷰를 받았고, 팀원들과 리서치·컨셉 도출·UI 디자인을 분담해 수행했습니다.

## 결과 및 회고

웨일만의 차별화된 기능을 새로운 온보딩 흐름으로 경험할 수 있는 TO-BE 프로토타입을 완성했습니다. 유저 리서치 이론을 실제 서비스 개선에 적용해 보는 경험이었고, 네이버 웨일 담당자 리뷰를 통해 실무 관점의 피드백도 받을 수 있었습니다.

![프로젝트 마무리](/images/how/whale-onboarding/23.png)
